set_options(options)[ソース]¶

オプションを設定します。

  • options : dict

    オプション名と値のdict型オブジェクト。

    • correct_mode : int

      マニュアル補正計測を開始する。値は計測回数

    • face_detect : bool

      人体検知を行うか (デフォルト: True)

    • hide_bg : bool

      動体検知による背景削除を行うか (デフォルト: True)

    • image_tab : bool

      OwhDevice.get_frame で画像を返すか (デフォルト: True)

    • manu_corr : float

      OwhMeta.manu_corr の値。 SDK利用側で取得・保存しておいた値をここで設定することで、 マニュアル補正計測後と同じ状態に復元されます。 (デフォルト: 0)

    • manu_corr_ref : float

      OwhMeta.manu_corr_ref の値。 SDK利用側で取得・保存しておいた値をここで設定することで、 マニュアル補正計測後と同じ状態に復元されます。

    • manual_body_temp : float

      基準値とする正式な体温計で計測した体温。 correct_mode でマニュアル補正計測を開始するときに設定が必要。 また、マニュアル補正計測の状態を復元するときは manu_corr / manu_corr_ref と同時に設定します。

    • obs_points : [[int, int], [int, int], [int, int]]

      OwhMeta.obs_temps で返される温度の座標。 3点まで指定可能で [[x1, y1], [x2, y2], [x3, y3]] の形式で設定します。 X座標・Y座標の範囲は 0 .. 119 です。 (デフォルト: [[60, 30], [60, 60], [60, 90]])

    • show_target : bool

      人体検知での温度取得位置を表示するか (デフォルト: True)

    • target_dist : int

      距離のよる補正の固定値(mm単位)。 use_dist = 2 のとき有効。

    • target_emiss : float

      測定対象の放射率 (デフォルト0.98)

    • temp_tab : bool

      OwhMeta.temp_tab を返すか (デフォルト: False)

    • use_dist : int

      距離による補正の動作

      • 0: 距離による補正をしません

      • 1: 距離センサの出力で補正(デフォルト)

      • 2: target_dist の値で補正

class OwhMeta[ソース]¶

ベースクラス: object

フレームに付随するデータを表します。

CE_NONE = 0¶

correct_error : 成功。

CE_TOO_HIGH = 1¶

correct_error : 検知した温度が高すぎます。 options の manual_body_temp と比較して、 マニュアル補正計測中に検知された体表温度が高すぎるとき返されます。

CE_TOO_LOW = 2¶

correct_error : 検知した温度が低すぎます。 options の manual_body_temp と比較して、 マニュアル補正計測中に検知された体表温度が低すぎるとき返されます。

EV_BODY_TEMP = 4¶

event_type : body_temp を検知しました。

EV_CORRECT = 2¶

event_type : 補正処理中に body_temp を検知しました。

EV_DIST_INVALID = 32¶

event_type : 物体が計測可能距離外に出ました。

EV_DIST_VALID = 16¶

event_type : 計測可能距離内に物体を検知しました。

EV_LOST = 8¶

event_type : 追跡していた動体を見失いました。

EV_NONE = 0¶

event_type : イベントなし。

S_ERROR = 1¶

status : エラーが発生して温度を取得できない状態。

S_INVALID_TEMP = 3¶

status : 温度データを取得できるが誤差が大きい状態。 電源ONの後、約20分間取り得えます。

S_NO_TEMP = 2¶

status : まだ温度データを取得できない状態。 電源ONの後、最初の数秒間取り得えます。

S_OK = 0¶

status : 通常の状態。
